# Service Request API

## Overview

A lightweight REST API for creating and listing service requests. It supports three request categories (Billing, Technical, Other), validates all input fields before saving, and stores requests in memory. Built with Python and FastAPI.

## How to Run

**Prerequisites:** Python 3.9 or higher.
Download from https://www.python.org/downloads/ if not already installed.

```bash
# 1. Clone the repository
git clone https://github.com/vsaisrujan/service-requests-api.git
cd service-requests-api
```

**2. Create and activate a virtual environment**

This keeps dependencies isolated and avoids conflicts with other Python projects.

On Mac/Linux:
```bash
python3 -m venv venv
source venv/bin/activate
```

On Windows (Command Prompt):
```bash
python -m venv venv
venv\Scripts\activate
```

On Windows (PowerShell):
```bash
python -m venv venv
venv\Scripts\Activate.ps1
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Start the API**

```bash
uvicorn main:app --reload
```

The API will be available at `http://127.0.0.1:8000/api/requests`.

Interactive docs (auto-generated by FastAPI): `http://127.0.0.1:8000/docs`

To stop the server, press `Ctrl+C`.

## Endpoints

| Method | Path                      | Description                      |
|--------|---------------------------|----------------------------------|
| POST   | `/api/requests`           | Create a service request         |
| GET    | `/api/requests`           | List all requests (newest first) |
| GET    | `/api/requests/{id}`      | Get a single request by ID       |

### Request Body (POST)

| Field          | Type   | Required | Notes                                        |
|----------------|--------|----------|----------------------------------------------|
| `title`        | string | Yes      | Non-empty, whitespace-only not accepted      |
| `description`  | string | Yes      | Non-empty, whitespace-only not accepted      |
| `category`     | string | Yes      | Must be exactly: `Billing`, `Technical`, or `Other` |
| `contactEmail` | string | Yes      | Must be a valid email format                 |

## Sample Requests

> **The API server must be running** (see How to Run above) before trying these commands.

> **Note for Windows users:** The curl examples below use single quotes which work on Mac/Linux. On Windows, use Postman or the interactive docs at `http://127.0.0.1:8000/docs` instead.

---

### Create a request (success)

```bash
curl -s -X POST http://127.0.0.1:8000/api/requests \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Payment failed",
    "description": "My last invoice shows an unexpected charge.",
    "category": "Billing",
    "contactEmail": "user@example.com"
  }'
```

**Response (201 Created):**
```json
{
  "id": "a3f1c2d4-89ab-4e12-b456-426614174000",
  "title": "Payment failed",
  "description": "My last invoice shows an unexpected charge.",
  "category": "Billing",
  "contactEmail": "user@example.com",
  "createdAt": "2026-05-24T10:30:00.123456+00:00"
}
```

> The `id` is a randomly generated UUID. The `createdAt` is a UTC timestamp. Both are assigned automatically by the server.

---

### Create a request (validation failure — missing field)

```bash
curl -s -X POST http://127.0.0.1:8000/api/requests \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Missing title field",
    "category": "Technical",
    "contactEmail": "user@example.com"
  }'
```

**Response (400 Bad Request):**
```json
{
  "detail": "title: Field required"
}
```

---

### Create a request (validation failure — invalid category)

```bash
curl -s -X POST http://127.0.0.1:8000/api/requests \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Test",
    "description": "Some description",
    "category": "Unknown",
    "contactEmail": "user@example.com"
  }'
```

**Response (400 Bad Request):**
```json
{
  "detail": "category: Input should be 'Billing', 'Technical' or 'Other'"
}
```

---

### Create a request (validation failure — invalid email)

```bash
curl -s -X POST http://127.0.0.1:8000/api/requests \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Test",
    "description": "Some description",
    "category": "Technical",
    "contactEmail": "not-an-email"
  }'
```

**Response (400 Bad Request):**
```json
{
  "detail": "contactEmail: value is not a valid email address: An email address must have an @-sign."
}
```

---

### List all requests

```bash
curl -s http://127.0.0.1:8000/api/requests
```

**Response (200 OK) — newest request appears first:**
```json
[
  {
    "id": "a3f1c2d4-89ab-4e12-b456-426614174000",
    "title": "Payment failed",
    "description": "My last invoice shows an unexpected charge.",
    "category": "Billing",
    "contactEmail": "user@example.com",
    "createdAt": "2026-05-24T10:30:00.123456+00:00"
  }
]
```

Returns an empty array `[]` if no requests have been created yet.

---

### Get a single request by ID

Copy the `id` value from a POST or GET response and use it in the URL below.

```bash
curl -s http://127.0.0.1:8000/api/requests/a3f1c2d4-89ab-4e12-b456-426614174000
```

**Response (200 OK):**
```json
{
  "id": "a3f1c2d4-89ab-4e12-b456-426614174000",
  "title": "Payment failed",
  "description": "My last invoice shows an unexpected charge.",
  "category": "Billing",
  "contactEmail": "user@example.com",
  "createdAt": "2026-05-24T10:30:00.123456+00:00"
}
```

**Response when ID does not exist (404 Not Found):**
```json
{
  "detail": "Request not found"
}
```

## Known Limitations

- **No persistence** — all requests are lost when the server restarts (in-memory only).
- **No pagination** — `GET /api/requests` returns all records.
- **No authentication** — any client can create or read requests.
- **No rate limiting** — not suitable for public exposure without a reverse proxy.
